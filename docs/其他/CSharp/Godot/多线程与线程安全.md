# Godot C# 进阶：多线程与 CallDeferred

#CSharp #Godot #Threading #Async

## 1. 致命红线：非线程安全

Godot 的 API (如 `AddChild`, `GetNode`, 修改 `Label.Text`) **绝大多数都不是线程安全的**。
这意味着：你**只能**在主线程（Main Thread）里动它们。

**❌ 错误示范 (必崩)**

```csharp
// 在后台线程跑任务
await Task.Run(() =>
{
    // 耗时计算...
    System.Threading.Thread.Sleep(2000);

    // 😱 试图在后台线程修改 UI -> 游戏崩溃！
    GetNode<Label>("ScoreLabel").Text = "计算完成";
});
```

---

## 2. 救命稻草：CallDeferred (延迟调用)

`CallDeferred` 的意思是：**“那个...我现在在后台忙，麻烦主线程你有空的时候（下一帧闲置时），帮我跑一下这个函数。”**

这是 Godot 提供的官方跨线程通信机制。

### A. 传统写法 (方法名反射)

```C#
// 这里的 "AddChild" 是方法名的字符串
// node 是参数
CallDeferred("add_child", node);
// 或者 C# 风格
CallDeferred(MethodName.AddChild, node);
```

### B. 现代写法 (Callable.From) —— ★★★ 推荐

Godot 4 引入了 `Callable`，配合 Lambda 表达式，写起来像 C# 原生一样舒服。

```C#
await Task.Run(() =>
{
    // 1. 在后台做繁重的计算/加载
    var resource = ExpensiveLoad();

    // 2. 切回主线程更新 UI 或 场景树
    // Callable.From 包装了一个 Action，让它在主线程稍后执行
    Callable.From(() =>
    {
        GetNode<Label>("ScoreLabel").Text = "加载完成";
        AddChild(resource);
    }).CallDeferred();
});
```

## 3. 实战场景：后台加载怪物

假设你要一次性生成 100 个怪物，如果在 `_Ready` 里直接生成，游戏会卡住 1 秒。

```C#
public void SpawnEnemiesAsync()
{
    Task.Run(() =>
    {
        var enemyScene = GD.Load<PackedScene>("res://Enemy.tscn");
        var enemies = new List<Node2D>();

        // 在后台线程实例化 100 个对象 (这只是内存操作，只要不 AddChild 就没事)
        for (int i = 0; i < 100; i++)
        {
            var enemy = enemyScene.Instantiate<Node2D>();
            enemy.Position = new Vector2(i * 10, 0);
            enemies.Add(enemy);
        }

        // 必须回主线程添加到树上
        Callable.From(() =>
        {
            foreach (var e in enemies)
            {
                AddChild(e);
            }
            GD.Print("100 个怪物已生成，丝毫没有卡顿！");
        }).CallDeferred();
    });
}
```
