# 手动测试指引

本文给人手动执行命令用。目标是快速验证 Godot 场景测试 runner、日志落盘、重试、视觉截图和收尾清理流程。

## 1. 进入项目根目录

```bash
cd /home/slime/Code/Godot/Games/MyGames/brotato-my
```

## 2. 验证 runner 可用

先做语法检查：

```bash
node --check .codex/skills/godot-scene-test/scripts/godot-scene-runner.mjs
```

预期：没有输出，退出码为 0。

再确认能列出测试场景：

```bash
node .codex/skills/godot-scene-test/scripts/godot-scene-runner.mjs list --filter VisualPreview
```

预期输出包含：

```json
{
  "scenes": [
    "res://Src/ECS/Test/GlobalTest/VisualPreview/VisualPreviewScene.tscn"
  ]
}
```

## 3. 验证 CLI 日志测试

执行一个短超时日志场景测试：

```bash
node .codex/skills/godot-scene-test/scripts/godot-scene-runner.mjs run res://Src/ECS/Test/SingleTest/Tools/Log/LogTest.tscn --build --attempts 2 --timeout 3000 --max-log-lines 4 --errors-only --log-dir Docs/测试/场景测试日志/runs
```

这个命令当前可能失败或超时。这里重点不是修 `LogTest.tscn`，而是验证 runner 行为。

重点看输出字段：

- `attemptsUsed`：应为 `2`，表示失败后重试了 2 次。
- `maxAttempts`：应为 `2`。
- `failureReason`：失败分类，例如 `TimedOut`。
- `firstError`：第一条关键错误。
- `logIndex`：本次日志索引路径。

如果输出里有：

```text
Docs/测试/场景测试日志/runs/2026-05-02/09-12-09/index.json
```

那本次运行目录就是：

```text
Docs/测试/场景测试日志/runs/2026-05-02/09-12-09
```

## 4. 检索日志

把 `<日期>/<时间>` 换成 `logIndex` 对应的目录：

```bash
rg -n -m 80 -C 3 "ERROR:|\\[ERROR\\]|\\[FAIL\\]|Exception|Cannot instantiate" Docs/测试/场景测试日志/runs/<日期>/<时间>
```

如果某个组件日志太多，继续叠加组件名或关键词：

```bash
rg -n -m 80 -C 3 "\\[LogTest\\]|Cannot instantiate|TimedOut" Docs/测试/场景测试日志/runs/<日期>/<时间>/**/combined.log
```

先看索引摘要：

```bash
rg -n "failed|status|failureReason|firstError|logFiles" Docs/测试/场景测试日志/runs/<日期>/<时间>/index.json
```

## 5. 什么时候用 headless CLI

AI 或人都应该按验证目标选择方式，不是固定必须 headless。

优先用 headless CLI：

- 逻辑流程。
- Entity / Component 生命周期。
- Data / DataKey。
- 碰撞语义。
- 异常、失败标记、日志输出。
- 批量回归。

不要只用 headless CLI：

- UI 布局是否重叠。
- 游戏实际画面、动画、材质、粒子。
- 摄像机效果。
- Godot 编辑器弹窗、报错窗口、插件界面。
- 需要截图给人判断的问题。

两者都需要时，先用 CLI 拿结构化日志，再用有界面 Godot 截图确认视觉表现。

## 6. 验证视觉截图

打开有界面 Godot 场景：

```bash
"${GODOT_BIN:-/home/slime/Code/Godot/GodotEngine/4.x/Godot_v4.6.2-stable_mono_linux_x86_64/Godot_v4.6.2-stable_mono_linux.x86_64}" --path . --scene res://Src/ECS/Test/GlobalTest/VisualPreview/VisualPreviewScene.tscn
```

窗口出现后，另开终端或切回终端执行：

```bash
flameshot gui -d 3000
```

执行后 3 秒内切到目标窗口并框选截图。截图结论和 CLI 日志结论分开记录。

## 7. 清理本轮日志

日志用完就删。只删除本轮 `logIndex` 所在的具体运行目录，不要删除整个 `runs/`。

```bash
rm -r -- Docs/测试/场景测试日志/runs/<日期>/<时间>
```

删除后确认目录不存在：

```bash
test ! -e Docs/测试/场景测试日志/runs/<日期>/<时间>
```

## 8. 常用完整流程

```bash

rg -n -m 80 -C 3 "ERROR:|\\[ERROR\\]|\\[FAIL\\]|Exception|Cannot instantiate" Docs/测试/场景测试日志/runs/<日期>/<时间>
rm -r -- Docs/测试/场景测试日志/runs/<日期>/<时间>
```
