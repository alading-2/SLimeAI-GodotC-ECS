
## 语言参考
- [C# 语言参考 | Microsoft Learn](https://learn.microsoft.com/zh-cn/dotnet/csharp/language-reference/)

**C# 的《语言参考》里埋着很多“历史包袱”和“废弃特性”**。如果你把它们当成平等的知识点去学，会严重拖慢你的效率。

### 核心区别：字典 vs. 教材

- **教程 (Tour/Fundamentals)** 像是 **《生存手册》**。
    
    - 它会告诉你：_“这把刀是用来切肉的（最常用），那把斧头是砍树的。”_
        
    - **侧重点**：教你 **Best Practice**（最佳实践），告诉你现在大家都在用什么。
        
- **参考 (Language Reference)** 像是 **《五金店存货清单》**。
    
    - 它会告诉你：_“这有一把生锈的锯子（C# 1.0 的老语法），这有一个核动力切割机（unsafe 指针操作），这有一个普通的螺丝刀。”_
        
    - **侧重点**：**Exhaustive**（详尽无遗）。它不负责告诉你哪个好用，只告诉你“我有这个功能”。
        

### ⚠️ 如果你要看《参考》，请务必开启“过滤器”

既然你倾向于看《参考》，我觉得完全可以。但为了不走弯路，我给你列一个 **“Godot 独立游戏开发避坑指南”**。

《语言参考》里有很多东西，你写游戏 **几乎永远用不到**，或者 **用了就是给自己找麻烦**。

#### 1. 看到这些章节，直接跳过（Skip）❌

| **章节/关键字**                  | **理由**                                                                     |
| --------------------------- | -------------------------------------------------------------------------- |
| **`unsafe` 代码 / 指针类型**      | 除非你在写高性能物理引擎底层，否则 Godot 开发不需要手动管理内存指针。                                     |
| **`fixed`, `stackalloc`**   | 同上，极度底层的内存优化，初期看了只会晕。                                                      |
| **`dynamic`**               | 既然选了 C# 这种静态语言，就尽量别用动态类型（性能差且容易报错），除非对接老旧 COM 组件。                          |
| **`goto`**                  | 结构化编程的大忌，写游戏逻辑千万别用。                                                        |
| **终结器 (Finalizers)**        | `~ClassName() { ... }` 这种写法在 Godot 里几乎不用，Godot 有自己的 `QueueFree()` 和内存管理机制。 |
| **`checked` / `unchecked`** | 处理整数溢出的，游戏里很少需要手动去控制这个。                                                    |
| **`volatile`**              | 多线程锁的高级用法，用 `lock` 或新的 `System.Threading.Lock` 就够了。                        |

#### 2. 看到这些章节，重点研读（Focus）✅

|**章节/关键字**|**理由**|
|---|---|
|**`class` vs `struct` vs `record`**|这是 C# 的根基，搞不清楚这三个的区别，游戏数据结构会一团糟。|
|**特性 (Attributes)**|Godot 全靠这个活着（`[Export]`, `[Signal]`, `[Rpc]`），必须精通。|
|**泛型 (Generics)**|`List<T>`, `Dictionary<T,K>`，写背包系统、对象池必用。|
|**委托与事件 (delegate/event)**|也就是 Godot 的信号（Signal），必须懂。|
|**模式匹配 (Pattern Matching)**|`switch` 表达式，C# 最现代、最爽的语法糖。|
|**空值安全 (Nullable)**|`?`, `??`, `??=`，防止 `NullReferenceException` 的神器。|

### 实际案例对比

比如 **“委托 (Delegate)”** 这个知识点：

- 看《教程》：
    
    它会告诉你：现在大家都用 Action 和 Func，配合 lambda 表达式 () => {} 写起来很快乐。
    
- 看《参考》：
    
    它会把 delegate 的老底都翻出来，包括 C# 1.0 时代的 BeginInvoke（异步调用模式，已过时）、MulticastDelegate 的内部实现机制。如果你花时间去研究 BeginInvoke，就是在学屠龙之技，因为现在都用 async/await 了。
