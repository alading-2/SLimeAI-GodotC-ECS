# Godot 异步编程 (Async/Await) —— 告别回调地狱

#CSharp #Godot #Async #Coroutine

## 1. 核心概念

- **同步**：你去排队买奶茶，点完单必须干等着，直到奶茶做好才能走。（卡住主线程，游戏画面卡死）
- **异步**：你点完单拿个号，去旁边玩手机（干别的），叫到号了再回来拿奶茶。（不卡主线程，游戏继续运行）

在 Godot 中，主要用它来处理 **时间等待**、**信号等待** 和 **资源加载**。

> **TS 开发者对照**：
> `async/await` 行为基本等同于 TS 的 `Promise`。
> `Task` ≈ `Promise`。

---

## 2. 基础语法

### 三大定律

1. 方法签名必须加 **`async`**。
2. 方法内部必须有 **`await`**，否则它就是个普通同步方法。
3. 返回值通常是 **`void`** (Godot 生命周期) 或 **`Task`** (如果是你要调用的工具函数)。

---

## 3. Godot 三大实战场景

### 场景一：简单的延时 (替代 Timer 节点)

这是最爽的功能。以前你要创建一个 Timer 节点，设置时间，连接 timeout 信号...
现在只需要一行代码。

```csharp
public async void OnSkillCast()
{
    GD.Print("开始施法...");

    // ⏳ 等待 1.5 秒
    // ToSignal 是 Godot 提供的魔法，它把信号变成了 Task
    // SceneTreeTimer 会受到游戏暂停和时间缩放的影响，非常适合游戏逻辑
    await ToSignal(GetTree().CreateTimer(1.5f), SceneTreeTimer.SignalName.Timeout);

    GD.Print("火球术发射！");
}
```

### 场景二：等待某个事件发生

比如：等待动画播放完毕，或者等待玩家按下任意键。

```C#
public async void PlayAttackAnim()
{
    var anim = GetNode<AnimationPlayer>("AnimationPlayer");
    anim.Play("Attack");

    // ⏳ 暂停这行代码的执行，直到动画播放器发出 "animation_finished" 信号
    // 此时游戏画面依然在动，不会卡死
    await ToSignal(anim, AnimationPlayer.SignalName.AnimationFinished);

    GD.Print("攻击动作做完了，可以移动了");
}
```

### 场景三：避免 `_Process` 里的状态机地狱

复杂的逻辑用 `async` 写会像剧本一样直观。

**❌ 以前 (状态机 + Timer):** 你需要定义枚举 `State.Idle`, `State.Attacking`，在 `_Process` 里判断，还要维护一个 `timer` 变量。

**✅ 现在 (线性代码):**

```C#
public async void ExecuteBossPattern()
{
    // 阶段 1：吼叫
    PlaySound("Roar");
    await ToSignal(GetTree().CreateTimer(2.0f), "timeout");

    // 阶段 2：冲锋
    TweenMoveToPlayer();
    // 等待 Tween 结束 (Tween 也可以 await！)
    await ToSignal(myTween, Tween.SignalName.Finished);

    // 阶段 3：休息
    PlayAnim("Tired");
    await ToSignal(GetTree().CreateTimer(3.0f), "timeout");

    // 循环
    ExecuteBossPattern();
}
```

## 4. 深度思考：`void` 还是 `Task`？

当你写一个异步方法时，返回类型怎么选？

- **`async void`**：
  - **适用**：**顶级入口**。比如 `_Ready()`, 按钮点击回调 `OnPressed()`, `_Process()`。
  - **原因**：Godot 引擎或者是 UI 事件系统不关心这个方法什么时候结束，只管触发它。
- **`async Task`**：
  - **适用**：**你自己写的工具方法**。
  - **原因**：如果你写 `void`，调用者没法 `await` 你。

**错误示范：**

```C#
// 定义
public async void WaitAndPrint() { // ❌ 应该用 Task
    await ToSignal(..., "timeout");
    GD.Print("Done");
}

// 调用
public async void _Ready() {
    WaitAndPrint(); // 😱 编译器不会报错，但它不会等待！
    GD.Print("Start");
    // 结果会是先打印 "Start"，后打印 "Done"，逻辑乱了。
}
```

**正确示范：**

```C#
// 定义
public async Task WaitAndPrint() { // ✅ 返回 Task
    await ...
}

// 调用
public async void _Ready() {
    await WaitAndPrint(); // ✅ 可以 await 了
    GD.Print("Start"); // 会在 Done 之后打印
}
```

---

## 5. 总结

1. **`await ToSignal(...)`** 是 Godot 异步的神技。
2. 用它来替代简单的 **Timer 节点** 和复杂的 **状态机**。
3. 记得：**工具函数返回 `Task`，回调函数返回 `void`。**
