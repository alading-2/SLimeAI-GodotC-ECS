# C# 泛型 (Generics) —— Godot 开发者的 "类型安全锁"

#CSharp #Godot #Generics

## 1. 一句话解释

泛型就是 **“参数化类型”**。
以前你写函数传的是**数据**（比如 `int`, `string`），现在你可以把**类型**（`T`）也当作参数传进去。

在 Godot 中，它的最大作用是：**消灭强制类型转换，让代码不再猜猜猜。**

---

## 2. Godot 里的三大实战场景

### 场景一：获取节点 (GetNode) —— 最常用

这是你每天都要写的代码。

**❌ 不用泛型 (旧时代)**

```csharp
// GetNode 返回的是 Node 类型，编译器不知道它是 Sprite2D
// 必须手动强转 (Sprite2D)，如果节点类型不对，运行时会报错
var player = (Sprite2D)GetNode("Player");
```

**✅ 使用泛型 (现代)**

```C#
// <T> 告诉编译器：我只要 Sprite2D，请直接给我这个类型
var player = GetNode<Sprite2D>("Player");
player.Texture = ...; // 此时有点号自动提示，因为编译器确定它是 Sprite2D
```

### 场景二：加载资源 (Load)

加载预制体 (`.tscn`) 或脚本资源。

```C#
// 明确告诉 Godot，我加载的是一个 PackedScene
var bulletScene = GD.Load<PackedScene>("res://bullet.tscn");

// 如果路径填错了文件类型（比如填了 .png），泛型虽然不能阻止加载，但能帮你理清变量类型
```

### 场景三：查找特定类型的子节点 (扩展方法)

假设你想写一个辅助函数，在某个节点下查找第一个某种类型的子节点。

```C#
// 定义一个泛型方法，<T> 是占位符
public T FindFirstChild<T>(Node parent) where T : Node
{
    foreach (var child in parent.GetChildren())
    {
        // 判断 child 是不是 T 类型
        if (child is T typedChild)
        {
            return typedChild;
        }
    }
    return null; // 或者 default(T)
}

// 使用：
var enemy = FindFirstChild<Enemy>(this);
var timer = FindFirstChild<Timer>(this);
```

## 3. 核心语法：约束 (Constraints)

当你写 `class MyManager<T>` 时，`T` 可以是任何东西（int, string, Button）。 但在 Godot 里，我们通常希望 `T` 必须是一个 **节点 (Node)**。

**关键字：`where`**

```C#
// 限制 T 必须继承自 Node (或者是 Node 本身)
// 只有这样，你才能在方法里调用 node.QueueFree() 等 Node 专属方法
public void Despawn<T>(T entity) where T : Node
{
    // ... 播放特效
    entity.QueueFree();
}
```

## 4. 泛型单例 (Singleton) —— 独立游戏神器

你肯定写过 `Global.Instance`。用泛型，你可以写一个 **“万能单例模板”**，以后任何管理器只要继承它，自动变成单例。

```C#
// 这是一个抽象的泛型单例基类
public abstract partial class Singleton<T> : Node where T : Node
{
    public static T Instance { get; private set; }

    public override void _Ready()
    {
        // 这一行有点绕，意思是把"我"赋值给 Instance
        Instance = this as T;
    }
}

// 使用方法：
// GameManager 自动拥有了 Instance 属性，不用再手写了
public partial class GameManager : Singleton<GameManager>
{
    public void GameOver() { ... }
}

// 调用：
GameManager.Instance.GameOver();
```

---

## 5. 总结

- **`<T>`**：读作 "Of T"，是一个类型的占位符。
- **主要用途**：`GetNode<T>()`, `GD.Load<T>()`, `GetComponent<T>()`。
- **好处**：**安全**（写错类型编译报错）、**方便**（不用写 `(Type)xxx` 强转）、**智能提示**。
