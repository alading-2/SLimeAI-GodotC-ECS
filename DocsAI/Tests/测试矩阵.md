# 测试矩阵

本文记录模块修改后优先运行哪些测试。当前是初版矩阵，后续随模块契约补齐。

## 当前验证基线（ECS-return）

当前回归旧 ECS 主线后，验证采用分层推进。下面是未来任务的最低阶梯，不表示本文件中的命令已在本次 reorientation change 中通过。

| 层级 | 目标 | 当前状态 |
| --- | --- | --- |
| L0 docs/path check | `rg` / `find` 检查入口、旧词分类、目标文件存在 | 当前文档治理 change 使用 |
| L1 build | 候选命令：`dotnet build` | 候选，必须实际执行后才能声明 PASS |
| L2 ECS core validation scene | Entity / Data / Event / SystemCore 等核心场景 | 候选场景见下方矩阵，命令参数需以真实 runner 为准 |
| L3 owner module validation | Ability / Damage / Movement / AI / Collision / UI 等 owner 模块专项场景 | 候选场景见下方矩阵 |
| L4 main scene smoke | 主场景或全局场景 smoke | 只能作为回归补充，不能替代专项验收 |

Godot scene pass claim 必须优先检查结构化 artifact：runner `index.json`、per-scene `result.json`、scene artifact，且 artifact 中 `expectedInputs`、`expectedObservations`、`passCriteria`、`failCriteria`、`artifactPath` 非空。仅 stdout `PASS`、`exitCode=0` 或“无 error”不能作为正确性证明。

## 批量回归

核心或跨模块修改后运行：

```bash
# .claude Skill 路径：
node .claude/skills/GodotSkill/scripts/godot-scene-runner.mjs run-many <相关场景1> <相关场景2> --build --continue-on-fail --attempts 2 --errors-only --log-dir .ai-temp/scene-tests/runs
# .codex Skill 路径：
node .codex/skills/godot-scene-test/scripts/godot-scene-runner.mjs run-many <相关场景1> <相关场景2> --build --continue-on-fail --attempts 2 --errors-only --log-dir .ai-temp/scene-tests/runs
```

一般不要默认全量跑所有测试；优先按上表选择相关场景并用 `run-many` 顺序执行。

只有核心或跨模块大改、发布前回归时，再运行：


## 收尾清理

```bash

```

只删除本轮命令返回的 `logIndex` 所在目录，不要删除整个 `runs/`。
