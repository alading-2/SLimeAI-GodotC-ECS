## 批量回归

核心或跨模块修改后运行：

```bash
# .claude Skill 路径：
node .claude/skills/GodotSkill/scripts/godot-scene-runner.mjs run-many <相关场景1> <相关场景2> --build --continue-on-fail --attempts 2 --errors-only --log-dir .ai-temp/scene-tests/runs
# .codex Skill 路径：
node .codex/skills/godot-scene-test/scripts/godot-scene-runner.mjs run-many <相关场景1> <相关场景2> --build --continue-on-fail --attempts 2 --errors-only --log-dir .ai-temp/scene-tests/runs
```

一般不要默认全量跑所有测试；优先按上表选择相关场景并用 `run-many` 顺序执行。

只有核心或跨模块大改、发布前回归时，再运行：

```bash
# .claude Skill 路径：
node .claude/skills/GodotSkill/scripts/godot-scene-runner.mjs run-all --build --continue-on-fail --attempts 2 --errors-only --log-dir .ai-temp/scene-tests/runs
# .codex Skill 路径：
node .codex/skills/godot-scene-test/scripts/godot-scene-runner.mjs run-all --build --continue-on-fail --attempts 2 --errors-only --log-dir .ai-temp/scene-tests/runs
```

## 收尾清理

完成分析、复验和总结后删除本轮具体运行目录：

```bash
rm -r -- .ai-temp/scene-tests/runs/<日期>/<时间>
```

只删除本轮命令返回的 `logIndex` 所在目录，不要删除整个 `runs/`。
