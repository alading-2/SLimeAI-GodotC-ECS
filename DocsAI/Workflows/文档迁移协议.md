# 文档迁移协议

本文定义 `Docs/`、`DocsAI/`、`.claude/skills/` / `.codex/skills/` 的职责边界。`Src/**/*.md` 已全部移除（除自动生成的 `InputManager.md`），源码入口统一由项目索引指向 .cs 文件。

## 目录职责

### Docs

面向人类阅读。放：项目愿景、架构设计、模块说明、使用指南、决策背景、历史方案和复盘。不放：AI 执行流水线细节、长任务恢复状态、Skill 内部触发规则、大量测试日志判定细节。

### DocsAI

面向 AI 执行。放：AI 开发流程、模块契约、测试矩阵、日志判定规则、长任务上下文、Skill 到详细文档的映射。不放：给人看的长篇故事化说明、与当前项目无关的学习笔记、未验证的临时想法。

### Skills

`.claude/skills/` / `.codex/skills/` — AI 工具的任务触发入口。`SKILL.md` 保持短，只放触发条件、边界、执行步骤、必读 DocsAI 链接、验证命令。详细模块知识放 `DocsAI/Modules/`，重复可靠流程放 `scripts/`。

## 文档分类规则

- 解释"为什么这样设计" → `Docs/`
- 解释"AI 执行任务必须怎么做" → `DocsAI/`
- 解释"触发这个任务后先做什么" → `.claude/skills/*/SKILL.md` / `.codex/skills/*/SKILL.md`
- 记录"本次迁移第几步、下一步是什么" → `DocsAI/ProjectState.md` 和对应计划目录
- 解释"这个源码目录有什么文件" → 不需要了，项目索引已覆盖

## 当前状态

- `Src/**/*.md` 已全部移除（除 `InputManager.md` 自动生成文件）
- 源码导航统一由 `Docs/框架/项目索引.md` 提供，直接指向 .cs 文件
- AI 执行入口：Skill → DocsAI/Modules/ → .cs 源码
- 人类阅读入口：Docs/README.md → 项目索引 → Docs/ 设计文档 或 .cs 源码

## 验证命令

```bash
find Src -name "*.md" ! -name "InputManager.md"  # 应为空
rg -n "Src/.*\.md" DocsAI Docs  # 应无失效引用
```
