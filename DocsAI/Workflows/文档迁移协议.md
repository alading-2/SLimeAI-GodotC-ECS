# 文档迁移协议

本文定义 `Docs/`、`DocsAI/`、`Src/**/*.md`、`.codex/skills/` 的职责边界和迁移规则。

## 目录职责

### Docs

面向人类阅读。

放：

- 项目愿景
- 架构设计
- 模块说明
- 使用指南
- 决策背景
- 历史方案和复盘

不放：

- AI 执行流水线细节
- 长任务恢复状态
- Skill 内部触发规则
- 大量测试日志判定细节

### DocsAI

面向 AI 执行。

放：

- AI 开发流程
- 模块契约
- 测试矩阵
- 日志判定规则
- 长任务上下文
- Skill 到详细文档的映射
- 外部框架研究协议

不放：

- 给人看的长篇故事化说明
- 与当前项目无关的学习笔记
- 未验证的临时想法，除非放入 `Archive/` 或 `Research/`

### Src/**/*.md

源码附近说明。

长期目标：

- 只保留短 README、局部 API 说明、源码目录职责。
- 长篇设计迁移到 `Docs/`。
- AI 执行契约迁移到 `DocsAI/`。

### .codex/skills

Codex 当前使用的任务触发入口。

长期目标：

- `SKILL.md` 保持短。
- 只放触发条件、边界、执行步骤、必须读的 DocsAI、验证命令。
- 详细模块知识放 `DocsAI/Modules/`。
- 重复可靠流程放 `scripts/`。

## 迁移步骤

1. 先建立新文档，不删除旧文档。
2. 在旧文档顶部增加新入口链接。
3. 更新 `Docs/框架/项目索引.md` 和 `DocsAI/INDEX.md`。
4. 更新相关 Skill 的文档路径。
5. 用 `rg` 检查旧绝对路径和失效引用。
6. 后续确认无引用后，再考虑删除或归档旧文档。

## 文档分类规则

- 解释“为什么这样设计” -> `Docs/`
- 解释“AI 执行任务必须怎么做” -> `DocsAI/`
- 解释“这个源码目录有什么文件” -> `Src/**/*.md`
- 解释“触发这个任务后先做什么” -> `.codex/skills/*/SKILL.md`
- 记录“本次迁移第几步、下一步是什么” -> `DocsAI/ProjectState.md` 和对应计划目录

## 验证命令

```bash
rg -n "<旧绝对路径正则>" Docs DocsAI Src .codex/skills
rg -n "DocsAI|文档迁移协议|AI开发闭环" Docs DocsAI .codex/skills
find DocsAI -maxdepth 3 -type f | sort
```

## 当前阶段默认策略

- 不全量移动旧文档。
- 不删除 `Src/**/*.md`。
- 不一次性重写所有 Skill。
- 优先建立入口、规则和索引，后续按模块迁移。
