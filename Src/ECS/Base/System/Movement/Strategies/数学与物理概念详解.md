# 移动策略数学入口

本文只保留源码旁数学索引。Movement 执行规则见 `DocsAI/Modules/Movement.md`，完整策略设计见 `Docs/框架/ECS/System/Movement/移动系统设计说明.md`。

## 坐标与角度

- Godot 2D：X 向右，Y 向下。
- 对外角度统一使用“度”。
- 项目语义：`0=右、90=下、180=左、正值顺时针`。
- 策略内部只在三角函数或 Godot API 需要时转换为弧度。

## 常用公式入口

- 方向单位化：`Vector2.Normalized()`。
- 垂直方向：根据策略需要选择顺 / 逆时针垂线。
- 正弦波：基础方向前进 + 垂直方向振荡。
- 贝塞尔：控制点曲线采样，追踪模式优先用模板重建剩余曲线。
- 抛物线 / 圆弧：由起点、终点和形状参数生成单段轨迹。
- 环绕：角运动 + 半径驱动合成。

## 修改提醒

- 不要把角度输入改成弧度。
- 不要在策略里直接移动节点。
- 不要把数值积分细节写进 Skill；AI 执行规则放 `DocsAI/Modules/Movement.md`。
