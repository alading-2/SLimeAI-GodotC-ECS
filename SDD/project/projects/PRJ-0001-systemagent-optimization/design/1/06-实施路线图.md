## 1. 总原则

实施顺序必须遵守：

1. 先跑通 SDD 手动闭环，再改 SystemAgent 默认规则。
2. 先把 SDD 从 `SDD/project/projects/PRJ-0001-systemagent-optimization/design/` 独立出去，再迁移设计文档。
3. 先修 hook 稳定性，再增加 hook 能力。
4. 先降低 workflow 入口成本，再考虑新增角色或 gate。
5. 先记录 worktree 策略，再自动化创建/清理。
6. 不把已有 OpenSpec change 强行迁移。

## 5. Phase 3：Workflow / Skill / Role / Subagent 分层重构

### Phase 3 目标

让 workflow 文件第一屏说明“怎么干活”，并明确 workflow 会调用哪些 capability skill、使用哪些 role、读写哪些 artifact，以及是否可选调用 subagent。

### Phase 3 任务

1. 修改 `Workspace/SystemAgent/Workflows/*.md` 的开头结构。
2. 增加 task_size 判断。
3. 增加 SDD required/optional/none 判断。
4. 区分 workflow entry skill 和 capability skill。
5. 新增或设计 `design-discovery` capability skill。
6. 新增或设计 `sdd-management` capability skill。
7. 在 NewFeature 中调用 `design-discovery` 和 `sdd-management`。
8. 增加 `DesignCritic` 条件角色设计。
9. 固化 selected workflow 输出格式。
10. 增加 Subagent Policy：默认只读，不做并行写代码。
11. 不新增 brainstorming hook，不新增逐问逐答交互器。

### Phase 3 验证

- 每个 workflow 前 30 行能看到触发、规模、调用 skill、使用 role、SDD、步骤、完成条件。
- 不复制 SDD 模板正文。
- 不引入新的大段 must-read。
- 新功能流程能输出风险、方案、推荐、必须确认项和默认假设。
- `DesignDiscovery` 和 `SDD Management` 能单独运行，也能被 workflow 调用。
- Subagent 只作为可选执行基座，不成为新的默认流程层。

## 10. 不建议做的事

### 10.1 不在 Phase 1 就写完整 CLI

先用手动 SDD 验证制度，再写工具。

### 10.2 不同时重写所有 SystemAgent 文档

先改入口和高频路径，再逐步清理协议和 gate。

### 10.3 不把 OpenSpec 目录直接删除

先退默认入口，再处理历史目录。

### 10.4 不新增只有 checklist 作用的 skill

能独立运行、被多个 workflow 复用、读写稳定 artifact 的能力可以做成 capability skill。

但单纯提醒项不要做成 skill，例如：

- 跑测试。
- 看 git status。
- 写总结。
- 不改同步副本。

### 10.5 不急着实现 subagent dispatcher

当前只建议 subagent 用于只读研究、独立评审和测试设计。

不建议现在做 tackle 式并行执行团队，因为还缺：

- 稳定 SDD 工作包格式。
- worktree 隔离策略。
- 文件 owner 协议。
- cleanup / timeout / retry 机制。
- 跨 Git 边界写操作保护。
