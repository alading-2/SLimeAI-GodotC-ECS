## 1. Goal

本轮只解决三个执行前未决问题：

```text
1. Common 工具放哪里。
2. NodeLifecycle 是否按 Runtime registry 迁移。
3. TargetSelector 是否保留旧 EntityTargetSelector.Query 兼容桥。
```

## 2. 用户已确认

用户 2026-06-07 确认：

1. Common 工具仍属于 Tools，工具都放在 `Src/ECS/Tools`。
2. `NodeLifecycle` 按此前建议处理：迁到 Runtime 语义，作为统一 Node lifecycle / registration / id / diagnostics registry。
3. `TargetSelector` 不做兼任，不保旧兼容桥；需要重构就完全重构。

## 3. Common Utilities 最终位置

最终目录采用：

```text
Src/ECS/Tools/CommonUtilities/
DocsAI/ECS/Tools/CommonUtilities/
```

原因：

- 符合用户“工具都放 `Src/ECS/Tools`”的目录规则。
- 又避免继续把杂项函数直接放在 `Src/ECS/Tools/CommonTool.cs` 或 `Src/ECS/Tools/` 根目录。
- 和现有 `Timer`、`ObjectPool`、`TargetSelector`、`Math` 一样，Common Utilities 是一个独立 Tool owner，不是所有工具的兜底垃圾桶。

准入规则：

- 只能放无明确 owner、无业务语义、无状态副作用的小工具。
- 每个 helper 必须有用途、输入、输出、副作用、热路径限制和测试入口。
- 能归入 Resource / Entity / Data / Event / Timer / TargetSelector / ObjectPool / Math / Capability / UI / Runtime 的函数，不能放 Common Utilities。
- `CommonTool.LoadPackedScene` 不属于 Common Utilities，必须迁入 `ResourceManagement` / `ResourceLoading`。
- 不新增 `CommonTool.SomeHelper()` 风格入口。

## 4. NodeLifecycle 最终位置

最终目录采用：

```text
Src/ECS/Runtime/NodeLifecycle/
DocsAI/ECS/Runtime/NodeLifecycle/
```

功能保留：

- Node 注册。
- Node 注销。
- Node id 管理。
- owner metadata。
- invalid cleanup。
- snapshot / diagnostics。

功能不保留：

- 不作为 Ability / AI / UI / Entity 的业务全局查询入口。
- 不让业务继续扫 `GetAllNodes()` / `GetNodesByInterface<T>()`。
- 不替代 EntityRegistry、UIManager、ComponentRegistrar、TargetQueryEngine。

通俗说：`NodeLifecycle` 是底层登记处，不是 gameplay 找目标、找 UI、找 Entity 的搜索工具。

## 5. TargetSelector 最终 cutover 口径

`TargetSelector` 执行型 SDD 不再设计兼容桥。

最终目标：

```text
旧入口删除：
  EntityTargetSelector.Query(query)
  PositionTargetSelector.Query(query)
  list-only TargetSelector facade

当前入口：
  TargetQueryEngine.QueryEntities(request)
  TargetQueryEngine.QueryPositions(request)
  TargetQueryResult<T>.Items
  TargetQueryResult<T>.Diagnostics
```

执行要求：

- Ability / AI / Feature / Tests 调用点一次性迁移到 `TargetQueryEngine`。
- DocsAI current 只教 `TargetQueryEngine` 和 `TargetQueryResult`。
- grep gate 必须确保业务路径不再使用旧 list-only API。
- 如果实现过程中出现本地临时中间态，不能作为完成状态提交；执行型 SDD 的完成定义是旧入口删除或至少不在 public/current API 中存在。

通俗说：以后 AI 找目标拿到的不是“一个 List”，而是“目标列表 + 为什么这样选的报告”。
