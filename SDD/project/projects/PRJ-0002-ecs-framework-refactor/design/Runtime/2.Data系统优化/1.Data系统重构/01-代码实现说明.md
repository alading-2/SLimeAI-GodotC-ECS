## 1. 实现总图

目标不是把 `Data` 做成纯数据库系统，而是把“AI 最难维护的字段定义”从 C# 静态 `DataMeta` 文件迁到 DataOS descriptor，再由运行时 catalog 消费。

```text
DataOS SQLite
  data_key_descriptor       字段定义 authoring 源
  业务表 / data_field        字段值 authoring 源
        │
        ▼
Generator + Validator
        │
        ▼
runtime_snapshot.json
  manifest
  descriptors              运行时字段定义输入
  records                  运行时字段值输入
        │
        ▼
RuntimeDataSnapshotLoader
        │
        ├── Build DataDefinitionCatalog
        ├── Register compute resolver
        └── Apply records to Data
        │
        ▼
Data
  Get / Set
  Default / Clamp / Options
  Modifier
  Computed cache
  Entity.Events property changed
```

实施目标不是让新旧通道长期共存。第一阶段可以读取旧 `DataMeta / DataRegistry / DataKey_*.cs` 生成一次性审计报告，但执行完成后必须删除旧运行时入口、旧配置输入路径和旧 Data 测试场景。

## 2. 新增核心模型

### 2.1 `DataValueType`

建议新增独立枚举，避免运行时直接依赖 `System.Type` 做所有判断。

```csharp
namespace SlimeAI.ECS.Base.Data;

public enum DataValueType
{
    String,
    Int,
    Float,
    Double,
    Bool,
    Vector2,
    ModifierList,
    ObjectRef
}
```

设计原因：

- **AI 友好**：descriptor 中只需要写 `float`、`int`、`bool` 等稳定字符串。
- **运行时稳定**：不在 JSON 里保存 `.NET Type` 名称。
- **可扩展**：后续可增加 `Vector2`、`Color`、`EnumString`。

### 2.2 `DataDefinition`

这是 `DataMeta` 定义职责的长期替代物，但不应把旧 `DataMeta` 的所有属性平铺成一个巨型对象。第一阶段可以先用一个类承载所有分层字段，代码结构上仍要区分 core、runtime policy、compute policy、migration policy 和 presentation。

```csharp
namespace SlimeAI.ECS.Base.Data;

public enum DataStoragePolicy
{
    Persisted,
    RuntimeState,
    RuntimeOnly,
    Computed,
    AuthoringBlob
}

public enum DataWritePolicy
{
    ReadWrite,
    LoaderOnly,
    SystemOnly,
    ComputedReadonly,
    DebugOnly
}

public enum DataRangePolicy
{
    None,
    Validate,
    ClampRuntime,
    RejectRuntime
}

public enum DataModifierPolicy
{
    None,
    Numeric,
    DebugOnly
}

public enum DataMigrationPolicy
{
    Default,
    Never,
    Always,
    ProfileOnly
}

public sealed class DataDefinition
{
    // 核心定义
    public required string StableKey { get; init; }
    public required DataValueType ValueType { get; init; }
    public string RuntimeTypeId { get; init; } = string.Empty;
    public required object? DefaultValue { get; init; }
    public string OwnerCapability { get; init; } = string.Empty;
    public string OwnerSkill { get; init; } = string.Empty;
    public DataStoragePolicy StoragePolicy { get; init; } = DataStoragePolicy.Persisted;
    public DataWritePolicy WritePolicy { get; init; } = DataWritePolicy.ReadWrite;

    // 运行时策略
    public DataRangePolicy RangePolicy { get; init; } = DataRangePolicy.None;
    public float? MinValue { get; init; }
    public float? MaxValue { get; init; }
    public DataModifierPolicy ModifierPolicy { get; init; } = DataModifierPolicy.None;
    public IReadOnlyList<DataAllowedValue> AllowedValues { get; init; } = Array.Empty<DataAllowedValue>();

    // 计算策略
    public string ComputeId { get; init; } = string.Empty;
    public IReadOnlyList<string> Dependencies { get; init; } = Array.Empty<string>();
    public IReadOnlyDictionary<string, string> ComputeParams { get; init; } = new Dictionary<string, string>();
    public bool IsComputed => StoragePolicy == DataStoragePolicy.Computed || !string.IsNullOrEmpty(ComputeId);

    // 迁移策略
    public DataMigrationPolicy MigrationPolicy { get; init; } = DataMigrationPolicy.Default;

    // 展示与工具元数据
    public string DisplayName { get; init; } = string.Empty;
    public string Description { get; init; } = string.Empty;
    public string OwnerDomain { get; init; } = string.Empty;
    public string UiGroup { get; init; } = string.Empty;
    public string ResetGroup { get; init; } = string.Empty;
    public string Unit { get; init; } = string.Empty;
    public string Format { get; init; } = string.Empty;
    public string IconPath { get; init; } = string.Empty;
}

public sealed class DataAllowedValue
{
    public required string Value { get; init; }
    public string Label { get; init; } = string.Empty;
}
```

关键差异：

- **`StoragePolicy` 区分持久化、运行时状态、运行时引用、computed 和 authoring blob**：例如 `Feature.Modifiers` 不是普通数值字段。
- **`WritePolicy` 明确写入权限**：computed 字段应是 `ComputedReadonly`，系统字段可设为 `SystemOnly`。
- **`RangePolicy` 不等于默认静默 clamp**：snapshot authoring 错误应优先被 validator 报告。
- **`ModifierPolicy` 替代裸 bool**：一次性审计阶段可从旧 `SupportModifiers` 映射为 `Numeric` 或 `None`，新字段必须直接写 policy。
- **`OwnerCapability / OwnerSkill` 进入定义**：AI 可以直接知道字段归属哪个系统和 skill。
- **`ComputeId` 替代 lambda**：DB 不保存 C# 函数，只保存计算语义 ID。
- **`Dependencies` 使用 stable key**：方便 validator 检查依赖是否存在。
- **`DisplayName / Description / UiGroup / Unit / Format` 保留为 presentation**：AI 和工具需要这些信息，但 `Data.Get/Set` 热路径不依赖它们。

## 3. 新增 `DataDefinitionCatalog`

Catalog 是运行时的字段定义索引，替代长期依赖 `DataRegistry.GetMeta(string key)` 的模式。

```csharp
namespace SlimeAI.ECS.Base.Data;

public sealed class DataDefinitionCatalog
{
    private readonly Dictionary<string, DataDefinition> _definitions = new(StringComparer.Ordinal);
    private readonly Dictionary<string, List<string>> _dependentsByKey = new(StringComparer.Ordinal);

    public void Register(DataDefinition definition)
    {
        ArgumentNullException.ThrowIfNull(definition);
        if (string.IsNullOrWhiteSpace(definition.StableKey))
        {
            throw new ArgumentException("DataDefinition.StableKey 不能为空。", nameof(definition));
        }

        if (!_definitions.TryAdd(definition.StableKey, definition))
        {
            throw new InvalidOperationException($"重复 DataDefinition：{definition.StableKey}");
        }

        for (var i = 0; i < definition.Dependencies.Count; i++)
        {
            var dependencyKey = definition.Dependencies[i];
            if (!_dependentsByKey.TryGetValue(dependencyKey, out var dependents))
            {
                dependents = new List<string>();
                _dependentsByKey[dependencyKey] = dependents;
            }

            dependents.Add(definition.StableKey);
        }
    }

    public bool TryGet(string stableKey, out DataDefinition definition)
    {
        return _definitions.TryGetValue(stableKey, out definition!);
    }

    public DataDefinition GetRequired(string stableKey)
    {
        if (TryGet(stableKey, out var definition))
        {
            return definition;
        }

        throw new KeyNotFoundException($"未注册 DataDefinition：{stableKey}");
    }

    public IReadOnlyList<string> GetDependentComputedKeys(string baseKey)
    {
        return _dependentsByKey.TryGetValue(baseKey, out var dependents)
            ? dependents
            : Array.Empty<string>();
    }
}
```

实现注意：

- **不要在 `Data` 每次 `Get/Set` 时扫描 descriptors**：启动时构建 catalog。
- **依赖反向索引在注册时建立**：用于 Set 后标脏 computed cache。
- **重复 key 必须 fail fast**：AI 写错 seed 或 generator 输出重复字段时不能静默覆盖。

## 5. Snapshot Loader

Loader 负责三件事：读取 JSON、构建 catalog、应用 records。

```csharp
namespace SlimeAI.ECS.Base.Data.RuntimeSnapshot;

public sealed class RuntimeDataSnapshotLoader
{
    private readonly DataComputeRegistry _computeRegistry;

    public RuntimeDataSnapshotLoader(DataComputeRegistry computeRegistry)
    {
        _computeRegistry = computeRegistry; // 计算字段 resolver 注册表
    }

    public RuntimeDataSnapshot LoadFromJson(string json)
    {
        ArgumentException.ThrowIfNullOrWhiteSpace(json);
        return JsonSerializer.Deserialize<RuntimeDataSnapshot>(json, SnapshotJsonOptions.Default)
            ?? throw new InvalidOperationException("runtime_snapshot.json 解析失败。");
    }

    public DataDefinitionCatalog BuildCatalog(RuntimeDataSnapshot snapshot)
    {
        var catalog = new DataDefinitionCatalog();
        for (var i = 0; i < snapshot.Descriptors.Count; i++)
        {
            var dto = snapshot.Descriptors[i];
            var definition = ConvertDescriptor(dto);
            ValidateComputeBinding(definition);
            catalog.Register(definition);
        }

        ValidateDependencies(catalog, snapshot.Descriptors);
        return catalog;
    }

    public DataApplyReport ApplyRecord(Data data, DataDefinitionCatalog catalog, RuntimeDataRecordDto record)
    {
        var report = new DataApplyReport(record.Table, record.Id);
        foreach (var pair in record.Fields)
        {
            var stableKey = pair.Key;
            var field = pair.Value;
            if (!catalog.TryGet(stableKey, out var definition))
            {
                report.AddError("snapshot.unknown_key", stableKey, "record field 没有对应 descriptor。");
                continue;
            }

            if (!DataValueConverter.TryConvert(field.Value, definition.ValueType, out var typedValue, out var error))
            {
                report.AddError("snapshot.wrong_type", stableKey, error);
                continue;
            }

            if (!data.SetUntyped(definition, typedValue))
            {
                report.AddError("snapshot.apply_rejected", stableKey, "Data 拒绝写入该值。");
                continue;
            }

            report.AppliedFieldCount++;
        }

        return report;
    }
}
```

Loader 必须保证：

- **descriptor 缺失报错**：record 里出现的每个 key 必须能在 catalog 找到。
- **type mismatch 报错**：record field 的 type 必须和 descriptor 的 `ValueType` 一致。
- **default 转换失败报错**：descriptor 默认值必须能转成目标类型。
- **computed resolver 缺失报错**：`StoragePolicy=Computed` 或 `ComputeId` 非空时，C# 必须注册 resolver。
- **Feature modifier 目标报错**：`Feature.Modifiers` 这类 `AuthoringBlob` 中的目标 key 必须存在且允许 modifier。
- **disabled capability 不进入 catalog**：由 DataOS generator/validator 先裁剪，loader 再防御性校验。

## 6. 修改 `Data`

现有 `Data` 的 modifier、computed cache、事件通知可以保留，但元数据来源从 `DataRegistry` 改为 `DataDefinitionCatalog`。

### 6.1 构造方式

```csharp
public sealed class Data
{
    private readonly DataDefinitionCatalog _catalog;
    private readonly Dictionary<string, object?> _values = new(StringComparer.Ordinal);
    private readonly Dictionary<string, object?> _computedCache = new(StringComparer.Ordinal);
    private readonly HashSet<string> _dirtyComputedKeys = new(StringComparer.Ordinal);

    public Data(Entity owner, DataDefinitionCatalog catalog)
    {
        Owner = owner; // 归属实体，用于事件通知
        _catalog = catalog; // 当前 profile 的字段定义 catalog
    }

    public Entity Owner { get; }
}
```

### 6.2 Typed API

长期推荐保留 `DataKey<T>` 作为可选类型安全 handle，但它不再是字段定义事实源。

```csharp
public readonly record struct DataKey<T>(string StableKey);

public T Get<T>(DataKey<T> key)
{
    return Get<T>(key.StableKey);
}

public bool Set<T>(DataKey<T> key, T value)
{
    return Set(key.StableKey, value);
}
```

`DataKey<T>` 的来源只能收敛到 generated thin handle：

- **执行早期**：可以临时手写极少量测试 handle，作为 TDD fixture。
- **正式路径**：由 snapshot descriptors/codegen 生成薄 handle，避免手写大量 key。
- **删除目标**：现有 `DataKey_*.cs` 不再作为字段定义文件保留。

### 6.3 字符串 API 降级为内部入口

```csharp
public T Get<T>(string stableKey)
{
    var definition = _catalog.GetRequired(stableKey);
    if (!DataValueConverter.IsCompatible<T>(definition.ValueType))
    {
        throw new InvalidOperationException($"Data.Get 类型不匹配：key={stableKey}, expected={definition.ValueType}, actual={typeof(T).Name}");
    }

    if (definition.IsComputed)
    {
        return (T)GetComputedValue(definition)!;
    }

    if (_values.TryGetValue(stableKey, out var value))
    {
        return (T)value!;
    }

    return (T)definition.DefaultValue!;
}

public bool Set<T>(string stableKey, T value)
{
    var definition = _catalog.GetRequired(stableKey);
    if (definition.WritePolicy == DataWritePolicy.ComputedReadonly)
    {
        return false;
    }

    if (!DataValueConverter.IsCompatible<T>(definition.ValueType))
    {
        return false;
    }

    if (!DataValueConverter.TryApplyRangePolicy(value, definition, out var finalValue, out _))
    {
        return false;
    }

    _values[stableKey] = finalValue;
    MarkDependentComputedDirty(stableKey);
    PublishPropertyChanged(stableKey, finalValue);
    return true;
}
```

约束：

- **业务代码不新增裸字符串调用**。
- **裸字符串 API 仅给 loader、一次性审计工具、测试 fixture 使用**。
- **后续可用 analyzer 或 grep gate 检查新增裸字符串访问**。

## 7. Compute resolver

DB 不保存 C# lambda。descriptor 只保存 `ComputeId`、`Dependencies` 和可选 `ComputeParams`，C# 注册 resolver。不要用 `FeatureSystem` 执行 computed：Feature 允许副作用和生命周期，而 computed 必须是无副作用、同步、可缓存的纯派生值。

### 7.1 Resolver 接口

```csharp
public interface IDataComputeResolver
{
    string ComputeId { get; }
    object? Compute(Data data, DataDefinition definition);
}
```

### 7.2 Registry

```csharp
public sealed class DataComputeRegistry
{
    private readonly Dictionary<string, IDataComputeResolver> _resolvers = new(StringComparer.Ordinal);

    public void Register(IDataComputeResolver resolver)
    {
        ArgumentNullException.ThrowIfNull(resolver);
        if (!_resolvers.TryAdd(resolver.ComputeId, resolver))
        {
            throw new InvalidOperationException($"重复 ComputeId：{resolver.ComputeId}");
        }
    }

    public IDataComputeResolver GetRequired(string computeId)
    {
        if (_resolvers.TryGetValue(computeId, out var resolver))
        {
            return resolver;
        }

        throw new KeyNotFoundException($"未注册 Data compute resolver：{computeId}");
    }
}
```

### 7.3 示例：属性加成计算

```csharp
public sealed class AttributeBonusComputeResolver : IDataComputeResolver
{
    public string ComputeId => "AttributeBonus";

    public object? Compute(Data data, DataDefinition definition)
    {
        if (definition.Dependencies.Count != 2)
        {
            throw new InvalidOperationException($"AttributeBonus 需要 2 个依赖：{definition.StableKey}");
        }

        var baseValue = data.Get<float>(definition.Dependencies[0]);
        var bonusValue = data.Get<float>(definition.Dependencies[1]);
        return MyMath.AttributeBonusCalculation(baseValue, bonusValue);
    }
}
```

对应 descriptor 可以是：

```json
{
  "stableKey": "Attribute.FinalHp",
  "valueType": "float",
  "defaultValue": "0",
  "storagePolicy": "computed",
  "writePolicy": "computed_readonly",
  "uiGroup": "Attribute.Computed",
  "computeId": "AttributeBonus",
  "dependencies": ["Attribute.BaseHp", "Attribute.HpBonus"],
  "minValue": 0,
  "rangePolicy": "validate",
  "modifierPolicy": "none"
}
```

这样 AI 改字段定义时只改数据：

- **新增一个同模式计算字段**：新增 descriptor 行，复用 `ComputeId=AttributeBonus`。
- **新增一种新公式**：才需要新增 C# resolver。
- **复杂公式仍由 C# 控制**：避免把代码塞进 DB。

## 8. `DataMeta` 一次性审计工具

第一阶段不设计运行时兼容层。可以新增一次性审计工具，把旧 `DataMeta` 转成 `DataDefinition` 草案，用于生成迁移报告和删除前缺口检查。该工具不应被 `Data` 运行时调用，执行完成后应删除。

```csharp
public static class LegacyDataAuditTool
{
    public static LegacyDataAuditEntry ToAuditEntry(DataMeta meta)
    {
        var draft = new DataDefinition
        {
            StableKey = meta.Key,
            ValueType = DataValueTypeMapper.FromSystemType(meta.Type),
            DefaultValue = meta.DefaultValue,
            StoragePolicy = meta.Compute != null ? DataStoragePolicy.Computed : DataStoragePolicy.Persisted,
            WritePolicy = meta.Compute != null ? DataWritePolicy.ComputedReadonly : DataWritePolicy.ReadWrite,
            RangePolicy = meta.MinValue.HasValue || meta.MaxValue.HasValue ? DataRangePolicy.ClampRuntime : DataRangePolicy.None,
            ModifierPolicy = meta.SupportModifiers == true ? DataModifierPolicy.Numeric : DataModifierPolicy.None,
            MigrationPolicy = meta.CanMigrate ? DataMigrationPolicy.Default : DataMigrationPolicy.Never,
            DisplayName = meta.DisplayName,
            Description = meta.Description,
            UiGroup = meta.Category?.ToString() ?? string.Empty,
            Unit = meta.IsPercentage ? "percent" : string.Empty,
            MinValue = meta.MinValue,
            MaxValue = meta.MaxValue,
            ComputeId = meta.Compute != null ? LegacyComputeIdMapper.GetRequired(meta.Key) : string.Empty,
            Dependencies = meta.Dependencies ?? Array.Empty<string>()
        };

        return new LegacyDataAuditEntry(meta.Key, draft);
    }
}
```

一次性审计工具用途：

- **迁移清单**：把旧 C# `DataMeta` 转成 descriptor 草案。
- **缺口检查**：某些 descriptor 缺失时，测试能指出缺口，而不是运行时静默 fallback。
- **删除门禁**：删除旧 `DataKey_*.cs` 前确认新 descriptor、调用点和 TDD 测试已经覆盖。

```text
descriptor.stable_key 非空且唯一
value_type 必须属于 string/int/float/double/bool/vector2/modifier_list/object_ref
默认值必须能按 value_type 转换
min_value <= max_value
storage_policy / write_policy / range_policy / modifier_policy / migration_policy 必须属于允许集合
storage_policy=computed 或 compute_id 非空时 compute_id 必须非空
compute_id 非空时 C# resolver manifest 必须存在对应 id
dependencies_json 中每个 key 必须存在 descriptor
Feature.Modifiers 的每个目标 key 必须存在，且 modifier_policy 允许 modifier
record field key 必须存在 descriptor
record field type 必须等于 descriptor.value_type
record field value 必须能按 descriptor.value_type 转换
```

## 10. 启动接入方式

旧 ECS 的 Entity 创建流程应从 profile 或 bootstrap 获得 `DataDefinitionCatalog`。

```csharp
public sealed class DataRuntimeBootstrap
{
    public DataDefinitionCatalog Catalog { get; private set; } = default!;
    public RuntimeDataSnapshot Snapshot { get; private set; } = default!;

    public void Initialize(string snapshotJson)
    {
        var computeRegistry = new DataComputeRegistry();
        RegisterFrameworkComputes(computeRegistry);

        var loader = new RuntimeDataSnapshotLoader(computeRegistry);
        Snapshot = loader.LoadFromJson(snapshotJson);
        Catalog = loader.BuildCatalog(Snapshot);
    }

    private static void RegisterFrameworkComputes(DataComputeRegistry registry)
    {
        registry.Register(new AttributeBonusComputeResolver());
        registry.Register(new PercentComputeResolver());
        registry.Register(new AttackIntervalComputeResolver());
    }
}
```

Entity 创建时：

```csharp
public Entity SpawnUnit(string unitRecordId)
{
    var entity = EntityManager.Spawn();
    entity.Data = new Data(entity, _dataRuntime.Catalog);

    var record = _dataRuntime.Snapshot.FindRecord("unit.enemy", unitRecordId);
    var report = _snapshotLoader.ApplyRecord(entity.Data, _dataRuntime.Catalog, record);
    if (report.HasErrors)
    {
        throw new InvalidOperationException(report.ToSummary());
    }

    return entity;
}
```

## 11. 完全重构步骤

### 11.1 第一步：建立 TDD 红灯测试和一次性审计

先写新 Data 系统测试，再新增一次性审计工具读取：

- 旧 `DataRegistry` 导出的 `DataMeta`。
- 新 snapshot descriptors。

输出：

```text
missing_in_snapshot
missing_in_csharp
type_mismatch
default_mismatch
range_mismatch
computed_missing_compute_id
unknown_dependency
old_path_reference
```

目标：先用 TDD 定义新系统行为，再用审计报告确认删除旧路径不会遗漏字段能力。

### 11.2 第二步：新增 descriptor-first runtime catalog

实现：

- `DataValueType`
- `DataDefinition`
- `DataDefinitionCatalog`
- `RuntimeDataSnapshotLoader`
- `DataComputeRegistry`

旧 `Data` 不应通过 adapter 长期同时支持 `DataMeta` 和 `DataDefinition`。执行早期如需桥接，只能在测试 fixture 或一次性审计工具中使用。

### 11.3 第三步：让 snapshot records 写入 `Data`

实现 record apply：

```text
record.fields[key]
  -> catalog.GetRequired(key)
  -> type convert
  -> range policy / allowed values validate
  -> data.SetUntyped(definition, value)
```

### 11.4 第四步：重建业务字段定义

按模块迁移：

1. Base / Unit
2. Attribute
3. Movement
4. Ability
5. Feature
6. AI / Test

每重建一组，运行审计报告和新 Data 行为测试，直到 descriptor 覆盖旧字段能力。

### 11.5 第五步：删除旧路径和旧测试

当 snapshot descriptor 覆盖完整后：

- 新增字段必须先写 DataOS descriptor。
- 禁止新增 `DataRegistry.Register(new DataMeta)`。
- 旧 `DataKey<T>` 只保留 stable key handle 或由 codegen 生成。
- `Data.Get<T>(string)` 降为内部 API。
- 删除或迁出 `SlimeAI/Data/Data/` 的 Data 输入职责。
- 删除 `SlimeAI/Data/DataNew/`。
- 重写 `SlimeAI/Src/ECS/Test/SingleTest/ECS/Data/`，不修补旧 `DataTestScene` / `TestDataKeyMapping`。

## 13. 不要做的事

- **不要让运行时直接查询 SQLite**：SQLite 是 authoring 源，不是游戏热路径依赖。
- **不要把 C# lambda 序列化进 JSON**：用 `ComputeId + resolver`。
- **不要保留旧 `Data/Data` 和 `DataNew` 输入路径**：完整重构完成后必须删除或迁出。
- **不要修补旧 Data 测试场景**：必须按新 Data TDD 矩阵重建。
- **不要继续把 `DataMeta` 当长期事实源**：否则 AI 仍然要维护大量 C# 字段定义。
- **不要把万能 `data_field` 当业务主表**：Unit / Ability / Feature 仍应优先走清晰业务表，再投影成 snapshot fields。
