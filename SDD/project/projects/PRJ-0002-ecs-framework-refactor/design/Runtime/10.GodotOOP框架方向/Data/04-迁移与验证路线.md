## 当前状态

本目录仍处于设计冻结阶段，不直接改运行时代码。

已确认：

- 不回退旧 `DataMeta`。
- 保留 DataOS / runtime snapshot / generated `DataKey<T>`。
- Data 不再是所有字段的唯一容器。
- Component / Profile / System 可以承载自己的运行时状态。
- Command 可以用，但采用 SlimeAI 风格 typed command/request + owner handler/pipeline。
- `AttackState / MoveMode / Velocity / AIState` 这类字段要按 authority / projection 重新审计。

## 迁移原则

### 1. 先审计字段，不先改代码

第一批审计字段：

```text
Health:
  CurrentHp, BaseHp, FinalHp, HpPercent, IsDead

Mana / Recovery:
  CurrentMana, FinalMana, FinalHpRegen, FinalManaRegen,
  IsDisableHealthRecovery, IsDisableManaRecovery

Attack:
  AttackState, AttackRange, AttackInterval,
  AttackWindUpTime, AttackRecoveryTime

Movement:
  MoveSpeed, FinalMoveSpeed, MoveMode, DefaultMoveMode,
  Velocity, VelocityOverride, VelocityImpulse,
  LastMoveDirection, MovementFacingDirection, IsMovementLocked

AI / Target:
  AIState, AIMoveDirection, AIMoveSpeedMultiplier,
  TargetNode, PatrolTargetPoint, PatrolWaitDone

Ability:
  AbilityCooldown, AbilityCurrentCharges, AbilityMaxCharges,
  AbilityCostType, AbilityCostAmount, FeatureIsActive, FeatureEnabled
```

每个字段必须补：

```text
authority
runtimeOwner
bindingPolicy
writeEntry
resetPolicy
modifier/computed eligibility
```

### 2. 先做纵切，不做全仓大迁移

推荐首个代码切片：

```text
Unit + Health + Damage + Recovery
```

原因：

- `CurrentHp` 是典型 Data authoritative。
- Damage/Heal 能验证 Command / owner API / Service Pipeline。
- HealthComponent 能验证 DataBinding mirror。
- RecoverySystem 当前同时存在 Event heal 和直接 `Data.Set(CurrentMana)` 两种写法，能暴露写入口不统一的问题。
- UI/Test/Log 都能观察 HP 变化。
- 对象池复用时 HP、IsDead、modifier source 容易验证。

### 3. Movement 和 AI 后置

Movement / AI 字段混合了共享输入、每帧内部状态和 debug projection，不适合做第一切片。

尤其是：

- `Velocity` 可能是跨系统协议，也可能只是内部每帧结果。
- `MoveMode` 可能是外部观察，也可能被业务当事实源写。
- `AIState` 更像 AI owner projection。

这些字段必须在 Health/Damage 样板稳定后再迁移。

## 建议后续 SDD

### SDD-A：Data Contract Routing Design

范围：

- 定义 descriptor 增量字段：`authority`、`runtimeOwner`、`bindingPolicy`、`writeEntry`、`resetPolicy`。
- 输出第一批字段审计表。
- 定义 generated C# contract 形状。
- 定义 Feature/Data manifest 对应关系。
- 不改 runtime。

验收：

- `Data/` 文档和 DocsAI Data 文档同步。
- 字段审计表覆盖 Health/Damage/Recovery 第一切片。
- SDD validate 通过。

### SDD-B：RuntimeRecordBinder + Health DataBinding

范围：

- 实现最小 `RuntimeRecordBinder`。
- 对 `CurrentHp / FinalHp / IsDead` 建立 Health binding。
- HealthComponent mirror 初始化和 DataChanged 增量同步。
- 对象池 acquire/release reset contract。
- Godot validation scene。

验收：

- 通过 DataOS validate。
- 通过 `dotnet build`。
- Godot headless scene 证明 Data -> Component mirror 和对象池复用不串值。

### SDD-C：Health/Damage/Recovery Mutation Boundary

范围：

- 定义 `DamageCommand` / `HealCommand` 或 typed request。
- 收口 `CurrentHp` / `CurrentMana` 写入口。
- DamageService / RecoverySystem 不再随意直接写共享字段，改走 owner API 或明确 owner 内部写。
- DataChanged / Event / Log / Test 统一验证。

验收：

- grep gate 分类 `Data.Set(CurrentHp|CurrentMana)`。
- 业务写入口有 owner。
- Health/Damage/Recovery scene 通过。

### SDD-D：Attribute Modifier Lane

范围：

- validator 校验 modifier 目标字段必须是 numeric Data authoritative 字段。
- `Feature.Modifiers` 到 DataModifier source 的授予/回滚。
- DataModifier 改动触发 DataChanged 和 DataBinding。
- 明确不支持 projection / enum / string / object_ref modifier。

### SDD-E：Movement / AI Projection Cleanup

范围：

- 审计 `AttackState / MoveMode / Velocity / LastMoveDirection / AIState / TargetNode`。
- 区分 Command、Event、Data authoritative、Component projection。
- 删除或 internal 化不该被外部写的 projection 入口。
- 补 Movement / AI validation scene。

## Grep gate 草案

文档冻结后的代码切片可先分类，不要求全部清零：

```bash
rg -n "Data\\.Set\\(GeneratedDataKey\\.(CurrentHp|CurrentMana|AttackState|MoveMode|Velocity|AIState)" Src/ECS -g '*.cs'
rg -n "Data\\.Get<.*GeneratedDataKey\\.(AttackState|MoveMode|Velocity|AIState|TargetNode)" Src/ECS -g '*.cs'
rg -n "TryAddModifier|AddModifier\\(" Src/ECS Data -g '*.cs'
rg -n "SetUntyped|TrySetUntyped|GetDiagnosticSnapshot|GetAll" Src/ECS Data -g '*.cs'
rg -n "Data 是唯一数据源|所有数据.*Data 容器" Src/ECS DocsAI SDD -g '*.cs' -g '*.md'
```

分类口径：

- owner 内部写：可保留。
- loader/debug/test 边界写：可保留但必须命名清楚。
- Component mirror 写：应改 DataBinding。
- projection 外部写：应收口。
- modifier 非数值字段：应禁止。
