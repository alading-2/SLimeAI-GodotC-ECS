## 1. `EntityId.cs`

目标代码形状：

```csharp
using System;

public readonly record struct EntityId(string Value)
{
    public static EntityId Empty { get; } = new(string.Empty);

    public static EntityId New() =>
        new(Guid.NewGuid().ToString("N"));

    public static EntityId From(string? value) =>
        string.IsNullOrWhiteSpace(value) ? Empty : new EntityId(value);

    public bool IsEmpty => string.IsNullOrWhiteSpace(Value);

    public override string ToString() => Value ?? string.Empty;
}
```

实现要求：

- 不提供 implicit string 转换。
- `EntityId.Empty` 是唯一空引用语义。
- 旧 `string id` 参数要在重构中逐个替换。
- DataOS / JSON 输出需要 string 时显式 `.Value` 或 `.ToString()`。

## 2. `EntityRegistry.cs`

目标代码形状：

```csharp
using Godot;
using System.Collections.Generic;

public sealed class EntityRegistry
{
    private readonly Dictionary<EntityId, Node> _nodesById = new();
    private readonly Dictionary<ulong, EntityId> _idsByInstanceId = new();

    public bool Register(EntityId entityId, Node node)
    {
        if (entityId.IsEmpty || node == null)
            return false;

        var instanceId = node.GetInstanceId();
        if (_nodesById.ContainsKey(entityId) || _idsByInstanceId.ContainsKey(instanceId))
            return false;

        _nodesById.Add(entityId, node);
        _idsByInstanceId.Add(instanceId, entityId);
        return true;
    }

    public bool Unregister(EntityId entityId)
    {
        if (!_nodesById.Remove(entityId, out var node))
            return false;

        _idsByInstanceId.Remove(node.GetInstanceId());
        return true;
    }

    public Node? Get(EntityId entityId) =>
        _nodesById.GetValueOrDefault(entityId);

    public EntityId GetEntityId(Node node) =>
        node == null ? EntityId.Empty : _idsByInstanceId.GetValueOrDefault(node.GetInstanceId(), EntityId.Empty);

    public IReadOnlyList<Node> Snapshot() =>
        new List<Node>(_nodesById.Values);
}
```

实现注意：

- `GetEntityId(Node)` 不能 fallback 到 `GetInstanceId().ToString()`。
- `Snapshot()` 必须复制，避免遍历中结构变更。
- `Unregister` 只清 registry，不负责 component、lifecycle、Data、Events 清理。销毁顺序由 destroy pipeline 统一编排。

## 4. `EntitySpawnPipeline.cs`

目标职责：

```text
1. validate request
2. allocate EntityId
3. create node
4. apply DataOS record
5. inject visual
6. apply transform
7. register entity
8. register components
9. attach lifecycle parent
10. activate pool
11. publish EntitySpawned
```

伪代码：

```csharp
public sealed class EntitySpawnPipeline
{
    private readonly EntityRegistry _registry;
    private readonly EntityNodeFactory _nodeFactory;
    private readonly EntityDataInitializer _dataInitializer;
    private readonly EntityVisualInitializer _visualInitializer;
    private readonly EntityTransformInitializer _transformInitializer;
    private readonly ComponentRegistrar _componentRegistrar;
    private readonly LifecycleTree _lifecycleTree;

    public SpawnResult<T> Spawn<T>(EntitySpawnRequest request) where T : Node, IEntity
    {
        var entityId = request.EntityId.IsEmpty ? EntityId.New() : request.EntityId;
        var create = _nodeFactory.Create<T>(request);
        if (!create.Success)
            return SpawnResult<T>.Fail(create.Error);

        var node = create.Node;
        if (!_dataInitializer.Apply(node, entityId, request, out var dataError))
        {
            _nodeFactory.Rollback(node, request);
            return SpawnResult<T>.Fail(dataError);
        }

        if (!_visualInitializer.Inject(node, request, out var visualError))
        {
            _nodeFactory.Rollback(node, request);
            return SpawnResult<T>.Fail(visualError);
        }

        _transformInitializer.ApplyBeforeRegister(node, request);

        if (!_registry.Register(entityId, node))
        {
            _nodeFactory.Rollback(node, request);
            return SpawnResult<T>.Fail($"Entity register failed: {entityId}");
        }

        _componentRegistrar.RegisterComponents(entityId, node);

        if (!request.LifecycleParentId.IsEmpty &&
            !_lifecycleTree.Attach(request.LifecycleParentId, entityId, request.ParentDestroyPolicy))
        {
            DestroyFailedSpawn(entityId, node, request);
            return SpawnResult<T>.Fail($"Lifecycle attach failed: {request.LifecycleParentId} -> {entityId}");
        }

        _nodeFactory.Activate(node, request);
        PublishSpawned(entityId, node);
        return SpawnResult<T>.Ok(entityId, node);
    }
}
```

硬规则：

- Relationship 绑定不在 spawn pipeline 中出现。
- 业务 DataKey 写入不在 Entity core 中出现。
- Lifecycle attach 失败必须让 spawn 失败，不允许 warn 后继续。
- 每个阶段返回可记录的 error，供测试和 observation 使用。

## 5. `EntityNodeFactory.cs`

负责：

- `ResourceManagement.Load<PackedScene>(typeof(T).Name, ResourceCategory.Entity)`
- `ObjectPoolManager.GetPool<T>(poolName)`
- `pool.Get(false)`
- 非池化实例 `scene.Instantiate<T>()`
- 添加到 `ParentManager.GetOrRegister(...)`
- 失败回滚 `QueueFree` 或归还池
- 最终 `Activate`

不负责：

- Data apply
- Component 注册
- Lifecycle attach
- 业务 owner 写入

对象池注意：

- 复用对象在 Activate 前不能启用碰撞参与宽相。
- Transform 应先设置，再 Activate。
- CharacterBody2D 的零速度 / `MoveAndSlide` 延迟逻辑保留在 transform/factory 协作中，但必须有测试。

## 6. `EntityDataInitializer.cs`

负责从 DataOS runtime snapshot 给实体写 Data：

```csharp
public sealed class EntityDataInitializer
{
    public bool Apply<T>(
        T entity,
        EntityId entityId,
        EntitySpawnRequest request,
        out string error) where T : Node, IEntity
    {
        var bootstrap = request.RuntimeDataBootstrap ?? DataRuntimeBootstrap.Default;
        var record = ResolveRecord(bootstrap, request);
        var report = bootstrap.ApplyToData(entity.Data, record);
        if (report.HasErrors)
        {
            error = report.ToSummary();
            return false;
        }

        if (!entity.Data.Set(GeneratedDataKey.Id, entityId.Value))
        {
            error = $"Entity id projection write failed: {entityId}";
            return false;
        }
        error = string.Empty;
        return true;
    }
}
```

规则：

- `GeneratedDataKey.Id` 是 EntityId 的 DataOS 字符串投影。
- 不再读 `DataKey.Id` 作为另一套入口；旧 `DataKey` 别名和 `DataKey<T> -> string` 隐式转换已删除。
- 业务系统需要身份时用 `IEntity.EntityId` 或 `EntityRegistry.GetEntityId(Node)`，不散读 `GeneratedDataKey.Id`。
- 如果 `GeneratedDataKey.Id` descriptor 不存在，重构必须先修 DataOS descriptor、snapshot 和 generated handle，而不是 fallback。

## 7. `ComponentRegistrar.cs`

当前 `RegisterComponents` 可以先迁入新类，但要改掉 Relationship 依赖。

目标伪代码：

```csharp
public sealed class ComponentRegistrar
{
    private readonly Dictionary<EntityId, List<Node>> _components = new();
    private readonly Dictionary<ulong, EntityId> _ownerByComponent = new();

    public void RegisterComponents(EntityId ownerId, Node entity)
    {
        var components = DiscoverComponents(entity);
        _components[ownerId] = components;

        foreach (var componentNode in components)
        {
            _ownerByComponent[componentNode.GetInstanceId()] = ownerId;
            if (componentNode is IComponent component)
                component.OnComponentRegistered(entity);
        }
    }

    public EntityId GetOwnerEntityId(Node componentNode) =>
        _ownerByComponent.GetValueOrDefault(componentNode.GetInstanceId(), EntityId.Empty);
}
```

迁移重点：

- `GetEntityByComponent` 改为 `ComponentRegistrar.GetOwnerEntityId -> EntityRegistry.Get`。
- `GetComponent<T>(entity)` 改为从 `_components[ownerId]` 查，不走 Relationship。
- `RemoveComponent` 只改 owner index 和 Godot Node，不移除 Relationship。
- `PrewarmComponentCache` 可保留，但由 `ComponentRegistrar` 拥有。

## 8. `LifecycleTree.cs`

目标实现：

```csharp
public sealed class LifecycleTree
{
    private readonly Dictionary<EntityId, LifecycleLink> _parentByChild = new();
    private readonly Dictionary<EntityId, List<LifecycleLink>> _childrenByParent = new();

    public bool Attach(EntityId parentId, EntityId childId, ParentDestroyPolicy policy, int priority = 0)
    {
        if (parentId.IsEmpty || childId.IsEmpty || parentId.Equals(childId))
            return false;
        if (_parentByChild.ContainsKey(childId))
            return false;
        if (WouldCreateCycle(parentId, childId))
            return false;

        var link = new LifecycleLink(parentId, childId, policy, priority);
        _parentByChild[childId] = link;
        GetChildrenList(parentId).Add(link);
        SortChildren(parentId);
        return true;
    }
}
```

销毁协作：

```text
Destroy(entityId)
  children = LifecycleTree.GetChildren(entityId)
  foreach child where DestroyPolicy == DestroyRecursively:
      Destroy(child.ChildEntityId)
  LifecycleTree.DetachAll(entityId)
  OwnedReferenceRegistry.NotifyDestroying(entityId)
  ComponentRegistrar.UnregisterComponents(entityId)
  EntityRegistry.Unregister(entityId)
  Data.Clear / Events.Clear
  NodeFactory.FreeOrReturnToPool(node)
```

顺序必须测试，不靠注释约定。
