### 2.1 Spawn 字段替换

旧字段：

```csharp

```

```csharp

```

业务 owner/source/target 不允许进 `EntitySpawnRequest`。

### 2.2 Register / Destroy 替换

旧：

```csharp
string id = entity.GetInstanceId().ToString();
NodeLifecycleManager.Register(entity);
EntityRelationshipManager.RemoveAllRelationships(id);
```

新：

```csharp
var entityId = request.EntityId.IsEmpty ? EntityId.New() : request.EntityId;
registry.Register(entityId, entity);
destroyPipeline.Destroy(entityId);
```

规则：

- `GetInstanceId()` 只允许作为 Godot node instance index，不允许成为 gameplay identity。
- `Destroy(Node)` 可以在重构中短暂作为内部 bridge，但 public runtime API 完成态必须是 `Destroy(EntityId)` 或 typed result。

## 3. Component 迁移

- `EntityManager_Component.cs` 多处写 `ENTITY_TO_COMPONENT`。
- `GetEntityByComponent` 通过 Relationship 反查 owner。
- `GetComponents` 通过 Relationship 查 component ids。

新实现：

public sealed class ComponentRegistrar
{
    private readonly Dictionary<EntityId, List<Node>> _componentsByOwner = new();

    public void UnregisterComponents(EntityId ownerId);
    public EntityId GetOwnerEntityId(Node componentNode);
    public IReadOnlyList<Node> GetComponents(EntityId ownerId);
}

迁移步骤：

2. GREEN：先把现有 scan 逻辑搬进 `ComponentRegistrar`，删除 Relationship 写入。
3. REFACTOR：把 `EndsWith("Component")` 标成临时兼容识别规则；后续只保留 `IComponent`。
4. 删除 `ENTITY_TO_COMPONENT` 调用和相关测试。

var spawn = entitySpawnPipeline.Spawn<ProjectileEntity>(new EntitySpawnRequest

## 6. Effect 迁移

当前命中：

- `EffectTool.Spawn` 手动实例化、手动 `GeneratedDataKey.Id = GetInstanceId().ToString()`。
- `EffectTool` 通过 `BindParentRelationships(... ENTITY_TO_EFFECT)` 绑定 host / owner。
- `EffectTool.DestroyByHost` 通过 `GetChildEntitiesByParentAndType(hostId, ENTITY_TO_EFFECT)` 查 effect。
- `EffectComponent` 通过 `EntityRelationshipTraversal.GetDirectParent` 找 host。

新流程：

| 需求 | 新表达 |
| --- | --- |
| 附着 effect 随 host 销毁 | `LifecycleParentId = hostId` |
| 独立 effect 随 source 或 scene scope 销毁 | `LifecycleParentId = sourceId` 或 scene lifecycle root |
| host 查询 effect | `EffectService` 内部 host index 或 host `SpawnedEffectIds` |
| effect source/target/ability | `EffectDataKeys.SourceEntityId / TargetEntityId / AbilityEntityId` |
| effect id | `EntityId` 分配后写 `GeneratedDataKey.Id` |

测试先行：

- host destroy 时 attached effect 按 lifecycle policy 销毁。
- DestroyByHost 不查 Relationship。
- effect component 读 host id 从 typed DataKey 或 service index，不查 parent chain。

## 7. Damage / Statistics 迁移

当前命中：

- `DamageInfo` 文档仍写统计归属通过 `EntityRelationshipTraversal.FindAncestorOfType<IUnit>(Attacker)`。
- `StatisticsProcessor` 通过 `GetAncestorChain(info.Attacker)` 统计 unit / weapon。
- `CritProcessor`、`LifestealProcessor` 通过 `FindAncestorOfType<IUnit>` 查 attacker/target。
- `DamageStatisticsSystem` 通过 `ENTITY_TO_ITEM` / ancestor chain 查物品和击杀者。

新结构：

```csharp
public readonly record struct DamageAttribution(
    EntityId SourceEntityId,
    EntityId DamageCreditEntityId,
    EntityId WeaponEntityId,
    EntityId AbilityEntityId,
    EntityId EffectEntityId,
    EntityId ProjectileEntityId);
```

迁移原则：

- `DamageInfo` 必须携带 `DamageAttribution`。
- `StatisticsProcessor` 只读 attribution，不查 lifecycle。
- `CritProcessor` / `LifestealProcessor` 的 attacker unit 从 `DamageCreditEntityId` 或明确 source owner DataKey 解析。
- `DamageStatisticsSystem` 的 item/weapon 统计从 `WeaponEntityId` / inventory service 读取，不通过 `ENTITY_TO_ITEM`。
- 缺 `DamageCreditEntityId` 是错误，不 fallback parent chain。

测试先行：

- projectile damage credit 给 player。
- weapon stats 记给 weapon。
- ability stats 记给 ability。
- lifecycle parent 改到 scene root 后统计仍正确。
- attribution 缺失时 validation fail。

## 8. Movement / Collision 迁移

当前命中：

- `MovementCollisionPolicy.FindAncestorOfType<IUnit>` 用 source parent chain 找 owner unit。
- `BoomerangStrategy.FindAncestorOfType<Node2D>` 用 parent chain 找 owner。

新入口：

```text
Movement source owner:
  read Projectile.SourceEntityId / Effect.SourceEntityId / Ability.OwnerEntityId
  then resolve owner unit by typed service/data

Boomerang owner:
  projectile.Data[ProjectileDataKeys.SourceEntityId]
```

规则：

- 阵营判断不允许沿 lifecycle tree 猜 owner。
- 如果 movement strategy 需要 owner position，spawn 时必须写 owner/source typed reference。
- 碰撞过滤读 explicit source owner，不能把 lifecycle parent 当阵营 owner。
