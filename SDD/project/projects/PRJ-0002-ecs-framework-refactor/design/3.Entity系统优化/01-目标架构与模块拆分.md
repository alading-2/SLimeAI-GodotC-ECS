# 目标架构与模块拆分

> 更新：2026-05-29
> 目标：把 Entity core 从“Godot Node 生命周期 + Data 初始化 + Relationship 图 + Component owner + Ability 管理 + Debug 查询”的混合体，重建为 AI-first 的小核心和明确 owner service。

## 0. 架构总图

目标结构：

```text
Entity Runtime Core
  EntityId
  EntityRegistry
  EntitySpawnPipeline
  LifecycleTree
  ComponentRegistrar
  EntityObservationDumper

Godot / Node Infrastructure
  EntityNodeFactory
  EntityVisualInitializer
  EntityTransformInitializer
  ComponentOwnerIndex

Capability-owned References
  Ability owner/list
  Projectile source/list
  Effect source/list
  Item / weapon owner/list
  UI binding registry

Damage Attribution
  DamageAttribution
  DamageInfo attribution fields
  StatisticsProcessor reads attribution only
```

核心原则：

- Entity core 只负责身份、注册、生命周期结构变更和 spawn 流程编排。
- Lifecycle parent 只表示生命周期，不表示业务归属。
- 业务引用属于各 capability，不属于 Entity core。
- Godot Node parent 只表示场景树组织，不表示 gameplay parent。
- Debug/AI 需要的关系图通过 observation dump 生成，不作为 runtime API。

## 1. Entity identity

### 当前问题

当前 ID 有三种来源：

- `Node.GetInstanceId().ToString()`
- `DataKey.Id`
- `GeneratedDataKey.Id`

`EntityRelationshipTraversal.ResolveEntityId` 会优先读 `DataKey.Id`，否则回退 Godot InstanceId。这个回退对临时调试有用，但对 AI-first 是坏入口：AI 无法判断某个字符串是不是稳定 runtime id，也无法知道 entity 是否注册过。

### 新设计

新增 typed identity：

```csharp
public readonly record struct EntityId(string Value)
{
    public static EntityId Empty { get; } = new(string.Empty);
    public static EntityId From(string? value) =>
        string.IsNullOrWhiteSpace(value) ? Empty : new EntityId(value);
    public bool IsEmpty => string.IsNullOrWhiteSpace(Value);
    public override string ToString() => Value ?? string.Empty;
}
```

规则：

- Runtime API 参数必须用 `EntityId`。
- `string` 只允许出现在 JSON / log / DataOS snapshot 边界。
- 手动注册 Godot Node 时必须显式分配 `EntityId`。
- 不允许从 `GetInstanceId()` 隐式生成 gameplay id。
- `GeneratedDataKey.Id` 存储的字符串必须等于 `EntityId.Value`。

### AI-first 效果

AI 看到 `EntityId` 就知道这是稳定 runtime identity；看到 string 就知道它不是 runtime API。这样可以用类型系统阻止“把资源 key、Node name、Data key、entity id 混传”的常见错误。

## 2. EntityRegistry

### 当前问题

当前 `EntityManager` 把注册、查询、销毁、对象池归还、关系清理、事件广播、Data 清空都放在一起。`NodeLifecycleManager` 也在底层维护注册状态，但上层 API 仍是 raw string。

### 新设计

`EntityRegistry` 是唯一注册表：

```csharp
public sealed class EntityRegistry
{
    private readonly Dictionary<EntityId, Node> _nodes = new();
    private readonly Dictionary<ulong, EntityId> _nodeToEntity = new();

    public bool Register(EntityId entityId, Node node);
    public bool Unregister(EntityId entityId);
    public Node? Get(EntityId entityId);
    public EntityId GetEntityId(Node node);
    public IReadOnlyList<Node> Snapshot();
    public void Clear();
}
```

规则：

- `Register` 失败时不自动覆盖旧实体。
- `GetEntityId(Node)` 找不到时返回 `EntityId.Empty`，不生成新 ID。
- `Snapshot()` 返回 copy，不暴露内部 dictionary。
- `Unregister` 不负责递归销毁，不调用业务 cleanup；销毁顺序由 `EntityLifecycleRuntime` 或 `EntityDestroyPipeline` 编排。

### 旧代码替换

| 当前入口 | 新入口 |
| --- | --- |
| `NodeLifecycleManager.Register(node)` | `EntityRegistry.Register(entityId, node)` |
| `NodeLifecycleManager.GetNodeById(string)` | `EntityRegistry.Get(EntityId)` |
| `EntityManager.GetAllEntities()` | `EntityRegistry.Snapshot()` + typed filter/service |
| `EntityManager.GetEntityById(string)` | `EntityRegistry.Get(EntityId)` |

## 3. EntitySpawnPipeline

### 当前问题

`EntityManager.Spawn<T>` 当前包含所有流程。最大风险不是代码长，而是每个阶段的失败语义和顺序不可单独测试。

### 新设计

`EntitySpawnPipeline` 只负责编排：

```text
Validate request
Allocate EntityId
Create node
Apply Data
Inject visual
Apply transform
Register entity
Register components
Attach lifecycle parent
Activate pool
Publish spawned event
```

建议阶段接口：

```csharp
public readonly record struct EntitySpawnRequest(
    string ResourceKey,
    object Config,
    EntityId EntityId,
    EntityId LifecycleParentId,
    ParentDestroyPolicy ParentDestroyPolicy,
    bool UseObjectPool,
    string? PoolName,
    Vector2? Position,
    float? Rotation,
    PackedScene? VisualSceneOverride);

public sealed class EntitySpawnPipeline
{
    public SpawnResult<T> Spawn<T>(EntitySpawnRequest request) where T : Node, IEntity;
}
```

`EntitySpawnRequest` 不包含：

- `ParentEntity`
- `AutoAddParentRelation`
- `ParentRelationTypes`
- 任意业务 owner/source/target 字段

业务引用在各 capability spawn tool 里写 DataKey，不进 Entity spawn request。

## 4. LifecycleTree

### 当前问题

`PARENT` 是字符串关系，但它实际承担单 parent、防环、递归销毁和 parent destroy policy。这些是生命周期树职责，不是通用 relation graph 职责。

### 新设计

```csharp
public readonly record struct LifecycleLink(
    EntityId ParentEntityId,
    EntityId ChildEntityId,
    ParentDestroyPolicy DestroyPolicy,
    int Priority);

public sealed class LifecycleTree
{
    public bool Attach(EntityId parentId, EntityId childId, ParentDestroyPolicy policy, int priority = 0);
    public bool Detach(EntityId childId);
    public EntityId GetParent(EntityId childId);
    public IReadOnlyList<LifecycleLink> GetChildren(EntityId parentId);
    public void DetachAll(EntityId entityId);
    public bool WouldCreateCycle(EntityId parentId, EntityId childId);
}
```

规则：

- 每个 child 至多一个 lifecycle parent。
- `EntityId.Empty` 不能作为 parent 或 child。
- self parent 拒绝。
- cycle 拒绝。
- children 查询返回 snapshot。
- destroy policy 是 typed field，不是 dictionary data。

## 5. ComponentRegistrar

### 当前问题

Component 注册走 `EntityRelationshipType.ENTITY_TO_COMPONENT`。这会让 Component owner 反查和 gameplay Relationship 混在一起。

### 新设计

Component owner 是内部 index：

```csharp
public sealed class ComponentRegistrar
{
    private readonly Dictionary<EntityId, List<Node>> _componentsByEntity = new();
    private readonly Dictionary<ulong, EntityId> _ownerByComponentInstanceId = new();

    public void RegisterComponents(EntityId ownerId, Node entityNode);
    public void UnregisterComponents(EntityId ownerId);
    public EntityId GetOwnerEntityId(Node componentNode);
    public IReadOnlyList<Node> GetComponents(EntityId ownerId);
}
```

规则：

- 注册识别策略应最终收敛为 `IComponent`，`EndsWith("Component")` 只允许在重构 SDD 中列为待删除项。
- `OnComponentRegistered(entity)` 前必须写入 owner index。
- 注销时先触发 component unregister，再清 index。
- Component owner index 不暴露为 Relationship。

## 6. OwnedReferenceRegistry

### 当前问题

`ENTITY_TO_PROJECTILE`、`ENTITY_TO_EFFECT`、`ENTITY_TO_ABILITY` 同时用于查询、清理和调试。它们不是 lifecycle parent，而是业务 owner-list。

### 新设计

业务 owner reference 走 typed DataKey：

```csharp
public readonly record struct EntityIdList(IReadOnlyList<EntityId> Ids)
{
    public static EntityIdList Empty { get; }
    public EntityIdList Add(EntityId id);
    public EntityIdList Remove(EntityId id);
}

public readonly record struct OwnedReferenceDescriptor(
    DataKey<EntityId?> ChildToOwnerKey,
    DataKey<EntityIdList> OwnerListKey);

public sealed class OwnedReferenceRegistry
{
    public void Register(OwnedReferenceDescriptor descriptor);
    public void NotifyDestroying(EntityId destroyedEntityId, Data destroyedData);
}
```

规则：

- Capability 初始化时注册 descriptor。
- Destroy child 时自动从 owner list 移除 child id。
- Destroy owner 时不通过 owner list 决定 child 生命周期；生命周期只看 `LifecycleTree`。

## 7. DamageAttribution

### 当前问题

伤害统计沿 `PARENT` ancestor chain 查 `IUnit` / `IWeapon`。这把“谁造成伤害”建模成“当前生命周期 parent 链长什么样”。

### 新设计

```csharp
public readonly record struct DamageAttribution(
    EntityId SourceEntityId,
    EntityId DamageCreditEntityId,
    EntityId WeaponEntityId,
    EntityId AbilityEntityId,
    EntityId EffectEntityId,
    EntityId ProjectileEntityId);
```

规则：

- `DamageInfo` 必须携带 attribution。
- `StatisticsProcessor` 不遍历 `LifecycleTree`。
- 缺失 `DamageCreditEntityId` 时，伤害统计失败并记录 validation error。
- 玩家、武器、技能统计分别读对应字段。

## 8. Observation

AI-first 不是把 runtime 做成通用图，而是把 runtime 状态导出成清楚的事实：

| Dump | 来源 |
| --- | --- |
| Entity registry dump | `EntityRegistry.Snapshot()` |
| Lifecycle dump | `LifecycleTree.GetChildren / GetParent` |
| Component owner dump | `ComponentRegistrar` |
| Typed reference dump | capability DataKey / owner list |
| Damage attribution trace | `DamageInfo.Attribution` |
| Spawn pipeline trace | 每个阶段 result |

Observation 可以用 string 字段序列化，但 runtime 内部仍使用 `EntityId`。

## 9. 非目标

本次不做：

- 不迁到纯 DOTS / archetype ECS。
- 不复制 Bevy / Flecs 完整关系系统。
- 不保留旧 Relationship runtime 做调试图。
- 不让 Entity core 知道 Ability / Projectile / Effect 的业务字段。
- 不把 Godot Node parent 当 lifecycle parent。
