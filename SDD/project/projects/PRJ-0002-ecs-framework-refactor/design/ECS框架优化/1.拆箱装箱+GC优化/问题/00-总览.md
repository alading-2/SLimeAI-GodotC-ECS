# ECS框架装箱拆箱与GC优化 - 总览

## 问题全景

SlimeAI ECS框架存在 **6大类、26个** 装箱拆箱/GC分配热点。根源是DataSlot用object?存储值类型，导致整个数据读写链路每帧产生上千次装箱。

## 严重度排名

| 级别 | 问题 | 每帧影响 | 预估分配/帧 |
|------|------|---------|------------|
| 🔴 CRITICAL | DataSlot object? 结构化装箱 | 每次Get/Set | 2000+次装箱+拆箱 |
| 🔴 CRITICAL | EventBus.EmitDynamic 反射+装箱 | 每次动态事件 | 反射+数组+装箱 |
| 🟠 HIGH | new Random() 热路径分配 | 每次TargetQuery | 1次堆分配 |
| 🟠 HIGH | List\<IEntity\> 无预设容量 | 每次TargetQuery | 3次List分配 |
| 🟠 HIGH | LINQ .Where().ToList() | 每次GetManualAbilities | List+迭代器 |
| 🟠 HIGH | DataConvert 返回object | 每次数据写入 | 每次写入装箱 |
| 🟡 MEDIUM | PropertyChanged事件 object? | 每次数据变更 | 2次装箱 |
| 🟡 MEDIUM | EventContext._result object? | 每次能力触发 | 1次装箱 |
| 🟡 MEDIUM | foreach Dictionary.Values | 每帧系统遍历 | Mono下可能分配 |
| 🟡 MEDIUM | ComponentRegistrar.ToArray() | 每次查询组件 | 1次数组分配 |
| 🟢 LOW | Subscription class | 每次订阅 | 1次堆分配 |
| 🟢 LOW | DataModifier.Source object? | 每次创建Modifier | 可能装箱 |

## 根因分析

### 核心架构问题：DataSlot用object?存储

```
Data.Set<float>("hp", 100f)          → (float)100f 装箱为 object
    ↓
DataSlot.SetValue(object?)           → 存储到 _value: object?
    ↓
Data.Get<float>("hp")                → 从 object? 拆箱回 float
    ↓
DataSlot.ApplyModifiers(object?)     → 拆箱→计算→重新装箱
    ↓
DataSlot.ConvertNumericToDefinitionType() → 显式 (object)(float)value
```

每帧每个实体的HP、速度、攻击力等数值读写都走这条路径。

### 数据流装箱链路图

```
写入路径（装箱）:
  Set<T>(key, T value) → TrySetUntyped(key, object?) → DataSlot.SetValue(object?)
  T value 装箱为 object

读取路径（拆箱）:
  Get<T>(key) → DataSlot.GetEffectiveValue() → (T)DataValueConverter.ConvertForRead()
  object? 拆箱回 T

修改器路径（拆箱→计算→重新装箱）:
  ApplyModifiers(object? baseValue) → TryGetNumeric(object, out double)
  → 计算 → ConvertNumericToDefinitionType(double) → (object)(float)result

变更通知路径（装箱）:
  Emit(new PropertyChanged(key, object? old, object? new)) → 两次装箱
```

## 优化路线图

### Phase 1: DataSlot类型分区（预计消除80%装箱）
- 将 DataSlot.Value: object? 替换为类型分区存储
- Dictionary<string, float> _floatSlots
- Dictionary<string, int> _intSlots
- Dictionary<string, bool> _boolSlots
- Dictionary<string, object?> _complexSlots（Vector2等复杂类型）
- 详见：[01-DataSlot结构化装箱.md](01-DataSlot结构化装箱.md)

### Phase 2: EventBus动态路径优化（预计消除15%分配）
- 缓存 EmitDynamic 的 MethodInfo + GenericMethod
- 或完全消除动态路径，改用类型注册表
- 详见：[02-EventBus装箱与反射.md](02-EventBus装箱与反射.md)

### Phase 3: ObjectPoolManager去反射（预计消除3%分配）
- 用接口多态替代反射调用
- 详见：[03-ObjectPoolManager反射调用.md](03-ObjectPoolManager反射调用.md)

### Phase 4: 热路径分配优化（预计消除2%分配）
- Random.Shared 替代 new Random()
- List预设容量 + 对象池
- LINQ替换为手动循环
- 详见：[04-LINQ热路径分配.md](04-LINQ热路径分配.md), [05-List与Dictionary分配.md](05-List与Dictionary分配.md)
